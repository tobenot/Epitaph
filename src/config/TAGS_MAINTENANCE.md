# 标签与技术栈分类系统维护指南 (Tags & Facets Maintenance)

本文档说明了 Epitaph 作品集中「标签系统」的设计逻辑，以及后续如何维护和增加新的标签与聚合分类。

## 1. 架构说明

为了解决自由标签过多、难以筛选的问题，系统采用了**双轨制**设计：

- **自由标签 (Tags)**：位于具体的项目文件内（如 `tags: ["VRChat", "模型展示", "鬼畜"]`）。用于描述项目细节，在详情页完整展示，首页卡片最多展示 3 个。它支持自由搜索。
- **聚合分类 (Facets)**：位于 `src/config/tagFacets.js` 中。通过设定匹配规则（`match`），自动捕捉分散的自由标签，将其归拢为一个可用于首页筛选的「技术栈/大类」（如 `LLM`、`UE5` 等）。

---

## 2. 如何维护聚合分类 (Facets)

如果你想在首页的「技术栈」筛选栏新增一个分类，或者修改现有分类的匹配规则，请修改 `src/config/tagFacets.js`。

### 配置文件示例
```javascript
export default [
  { 
    id: 'llm',           // 路由 URL 上的标识符 (如 /?facet=llm)
    label: 'LLM',        // 首页筛选按钮显示的文本
    match: [             // 匹配规则数组
      'LLM', 
      'Deepseek', 
      'AI大世界范式Ⅰ', 
      'AI大世界范式Ⅱ', 
      'AI大世界尝试'
    ] 
  },
  // ... 其他配置
]
```

### 匹配规则说明 (match)
- **模糊匹配**：如果项目包含的自由标签中，**包含了** `match` 列表里的任何一个词（或者反过来，`match` 词包含了标签），就会被归入这个 Facet。
- **大小写敏感度**：目前的匹配区分大小写。如果需要同时匹配 `ue5` 和 `UE5`，请注意标签书写的一致性。
- **自动隐藏**：如果某个 Facet 规则在目前的所有项目中没有命中任何一个项目（计数为 0），它会在首页**自动隐藏**，不会占用页面空间。

---

## 3. 如何为项目添加/修改自由标签

日常发布新作品或修改旧作品时，只需在 `src/config/projects/` 下的对应项目 `.js` 文件中修改 `tags` 字段即可。

```javascript
// src/config/projects/xxx.js
export default {
  // ...
  tags: ["C++", "独立游戏", "你的新标签"], 
  // ...
}
```

### 标签命名建议
1. **尽量收敛核心词**：像 `"组件化敌人分部件破坏以及影响行动"` 这种长句，建议把长句放入 `descriptionKey` 或 `developerNotesKey` 中，标签简化为 `"组件化敌人"`。
2. **复用词汇**：如果想让新标签能被现有的「技术栈」过滤面板捕捉到，请尽量使用已经在 `tagFacets.js` 中存在或能被 `match` 规则命中的词汇（例如写 `UE5` 而不是 `Unreal Engine 5`，除非你去更新 `match` 数组）。

---

## 4. 搜索逻辑 (Search)

用户在首页搜索框输入内容时，会匹配以下字段：
1. `titleKey` (标题)
2. `descriptionKey` (简介)
3. `tags` (所有自由标签)
4. `engine` (使用引擎)
5. `genres` (游戏类型，如果有)
6. `themes` (主题，如果有)

不需要针对搜索系统做额外的维护，只要你在项目中正确填写了以上字段，用户就能搜索到。
